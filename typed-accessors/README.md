# Typed Accessors

Read Parquet's **logical types** with [Hardwood](https://hardwood.dev)'s typed Row API accessors —
dates, times, timestamps, decimals, UUIDs, intervals, half-precision floats, and JSON/BSON
payloads — each decoded straight to its natural Java type. Uses tiny hand-built Parquet fixtures
bundled on the classpath, so it runs offline with no download and no compression codec.

## What you'll learn

- Decode the everyday logical types: `getDate` → `LocalDate`, `getTime` → `LocalTime`,
  `getTimestamp` → `Instant`, `getDecimal` → `BigDecimal`, `getUuid` → `UUID`.
- Tell **UTC** timestamps (`getTimestamp` → `Instant`, a point on the global timeline) from
  **local** ones (`getLocalTimestamp` → `LocalDateTime`, a zone-free wall clock) — and see the
  accessors **fail fast** when you ask for the wrong one.
- Read `getInterval` → `PqInterval` (`months` / `days` / `milliseconds` kept separate on purpose).
- Read FLOAT16 through `getFloat`, with `65504`, infinity, and NaN surviving the widen to `float`.
- Recognize the BYTE_ARRAY-backed types — **JSON**, **BSON** — and the **NULL** type from the
  schema, via `ColumnSchema.logicalType()`, then read the raw payloads with `getString`/`getBinary`.

## Run it

No setup beyond a JDK 21+. Pick either Maven or Docker.

**Maven** (from this folder, using the bundled Maven wrapper):

```shell
cd typed-accessors
./mvnw -q compile exec:java
```

**Docker:**

```shell
cd typed-accessors
docker compose run --rm --build typed-accessors
```

The sample Parquet files live on the classpath (`src/main/resources`), so there is nothing to
download and no `data/` folder. They are Hardwood's own test fixtures, generated by its PyArrow
data generator
[`tools/simple-datagen.py`](https://github.com/hardwood-hq/hardwood/blob/v1.0.0.CR2/tools/simple-datagen.py)
and checked into the Hardwood source tree under
[`core/src/test/resources`](https://github.com/hardwood-hq/hardwood/tree/v1.0.0.CR2/core/src/test/resources)
at the `v1.0.0.CR2` tag (each file linked from
[`Datasets.java`](src/main/java/dev/hardwood/examples/typedaccessors/Datasets.java)).

## How it works

[`Main.java`](src/main/java/dev/hardwood/examples/typedaccessors/Main.java) is short and linear, one
method per type group — start there. [`Datasets.java`](src/main/java/dev/hardwood/examples/typedaccessors/Datasets.java)
loads the bundled fixtures into memory.

- **Physical type vs. logical type.** Parquet stores a few physical types (INT32, INT64,
  BYTE_ARRAY, FIXED_LEN_BYTE_ARRAY) and tags them with logical types for meaning. `birth_date` is
  an INT32 tagged `DATE`; `profile_json` and `bson_payload` are both BYTE_ARRAY, separated only by
  their `JSON` / `BSON` annotation. The accessor decodes the physical bytes per the annotation.
- **Strict accessors.** A typed accessor decodes to exactly one Java type and refuses the rest:
  `getLocalTimestamp` on a UTC column throws `IllegalStateException` rather than inventing a zone.
  This "fail early" stance is what keeps a zone-less reading from being mistaken for an instant.
- **`PqInterval` keeps fields apart.** An interval's `months`, `days`, and `milliseconds` are not
  collapsed into one another, because a month and 30 days are not the same span on a calendar.
- **Recognizing a type from the schema.** `getFileSchema().getColumn(name)` returns a
  `ColumnSchema`; its `type()` is the physical type and `logicalType()` is the annotation (a sealed
  type you can `switch` over). That is how you spot JSON, BSON, FLOAT16, and the all-null NULL type.
- **No codec needed.** The bundled fixtures are uncompressed, so unlike the taxi examples this one
  declares no compression-codec dependency (see [`pom.xml`](pom.xml)).
